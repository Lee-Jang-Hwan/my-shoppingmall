# E2E 수동 점검 체크리스트

> Phase 6: 전체 사용자 플로우 E2E 점검을 위한 수동 체크리스트

## 📋 점검 가이드

### 점검 환경 설정

1. **개발 서버 실행**
   - 터미널에서 `pnpm dev` 실행
   - 브라우저에서 `http://localhost:3000` 접속 확인

2. **점검 전 준비사항**
   - Clerk 테스트 계정 준비 (회원가입용 이메일)
   - Supabase 대시보드 접근 권한 (사용자 동기화 확인용)
   - 테스트 상품 데이터 준비 (어드민에서 상품 등록)
   - 테스트 결제용 카드 정보 (Toss Payments 테스트 모드)

3. **점검 방법**
   - 각 항목을 순서대로 진행
   - 문제 발견 시 "발견된 이슈" 섹션에 기록
   - 체크박스(`- [ ]`)를 완료 시 `- [x]`로 변경

## 1. 회원가입 및 로그인 플로우 점검

### 1.1 Clerk 회원가입 프로세스 검증

**실행 절차:**
1. 홈페이지에서 "로그인" 또는 "회원가입" 버튼 클릭
2. Clerk 회원가입 폼 표시 확인
3. 이메일 주소 입력 (예: `test@example.com`)
4. 비밀번호 입력 및 확인
5. "회원가입" 버튼 클릭
6. 이메일 인증 완료 (테스트 환경에서는 즉시 완료 가능)
7. 회원가입 완료 후 홈페이지로 리다이렉트 확인

**예상 결과:**
- [ ] Clerk 회원가입 폼이 정상 표시됨

## 2. 상품 조회 및 탐색 플로우 점검

### 2.1 홈페이지 모든 섹션 표시 확인

**실행 절차:**
1. 홈페이지(`/`) 접속
2. 페이지 전체 스크롤하며 각 섹션 확인:
   - 카테고리 진입 섹션
   - 기획 상품 섹션 (PROMOTION)
   - 최신 상품 섹션
   - 인기상품 섹션 (BEST)
   - 디자인 콜라보 섹션 (COLLABORATION)

**예상 결과:**
- [ ] 카테고리 섹션이 표시됨 (상품이 있는 카테고리만 표시)
- [ ] 각 카테고리에 상품 개수가 정확히 표시됨
- [ ] 기획 상품 섹션이 표시됨 (프로모션 상품이 있는 경우)
- [ ] 최신 상품 섹션이 표시됨 (최대 12개)
- [ ] 인기상품 섹션이 표시됨 (판매 데이터가 있는 경우, 최대 8개)
- [ ] 디자인 콜라보 섹션이 표시됨 (콜라보 상품이 있는 경우, 최대 6개)
- [ ] 각 섹션의 제목이 정확히 표시됨
